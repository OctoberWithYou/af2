## 模型调用代理转发项目

诉求：我在内网有一些模型的 key，只有内网能访问，但是我的内网 windows 可以访问外网，我有台服务器，我计划用服务器做转发，使得外网可以通过访问服务器进而访问和使用内网的模型

## 实现方式

1. 语言使用 Java17 编码
2. 代理（agent）
   1. 独立模块 ✅ 已完成
   2. 可以转发 AI 访问请求，实际就是转发 http 或 https ✅ 已完成
   3. 可连接服务（server）构建一条长连接 ✅ 已完成
      1. 长连接使用 TCP，支持 SSL/TLS 加密，建议实现方式是使用 webSocket 协议，以避免中间人截获 ✅ 已完成
3. 服务（server）
   1. 独立模块 ✅ 已完成
   2. 部署在公网环境 ✅ 已完成
   3. 可接受代理 agent 的注册，并 1 对 1 ✅ 已完成
   4. 注册成功后可对外提供服务，使得用户可访问该服务器，转发 http 或 https 请求，以使用户认为自己在访问服务器，但实际上请求被转发到代理，由代理再进行转发 ✅ 已完成
   5. 用户访问前要先进行鉴权登录，防止中间人截获信息，鉴权可简单设计内置一个管理员用户 admin/Sys_ljc_123 ✅ 已完成
4. 部署（deploy）
   1. 独立模块 ✅ 已完成
   2. 作用是可以通过该模块对软件进行可视化的一键部署 ✅ 已完成
      1. 可一键部署代理 ✅ 已完成
      2. 可一键部署服务 ✅ 已完成
   3. 使用前后端分离技术 ✅ 已完成
      1. 后端 spring ✅ 已完成
      2. 前端 vue ✅ 已完成
      3. 部署要保证高易用性 ✅ 已完成
5. 打包
   1. 将软件打包，软件将包含 git tag desc 方便版本管理 ✅ 已完成
   2. 软件将收纳代理、服务、部署 ✅ 已完成（packageDistribution 任务）

## 规范

1. 代码风格务必高要求，所有功能必须有接口，接口内包含详尽注释，必须先设计，再评审，根据代码仓现状回顾评审，再编码
2. 所有新功能必须添加测试用例，并且需要在该模块添加一个测试清单，每次特性新增必须新增用例看护
3. 修改代码必须跑通原有测试用例，除非需要修改用例预期，也要保证用例修改后通过，代码才能上库
4. 定期提交 git，必须保证提交时黑盒用例的代码覆盖率达到 90%，并且所有用例全都通过
5. 所有问题包括优化，涉及到改项目文件的统一，若无需求跟踪统一任务是 bug
   1. bug 需要确定优先级、影响程度
   2. bug 出现时必须优先解决 bug，而后才开发需求
   3. 记录和审核 bug 数量，特性、修改引入 bug 过多需要回溯
   4. bug 修复后需要针对 bug 场景就行扩展测试，增加用例看护
6. 所有代码提交必须注明是特性还是问题，并且记录清单到项目中，定期回顾未解决的问题和待优化的特性
7. 所有与 ai 的对话全部存放在文件中，文件大小超过 10M 则切换文件
8. 以上规范必须严格遵守，写入本 ai 模型要求的规范文件中，每次修改项目文件务必要读取规范 review 审核

### 快速启动
```bash
# 构建项目
./gradlew build

# 运行测试
./gradlew test

# 生成代码覆盖率报告
./gradlew jacocoTestReport

# 打包分发版本
./gradlew packageDistribution

# 启动 Server
./gradlew :server:bootRun

# 启动 Agent
./gradlew :agent:bootRun

# 启动 Deploy（含前端页面 http://localhost:8090/api/deploy/）
./gradlew :deploy:bootRun
```

### 分发内容
执行 `./gradlew packageDistribution` 后，构建产物位于 `build/distribution/`：
```
build/distribution/
├── agent/           # Agent 模块 JAR 和配置
├── server/          # Server 模块 JAR 和配置
├── deploy/          # Deploy 模块 JAR 和配置
└── bin/             # 启动脚本
    ├── start-agent.sh
    ├── start-server.sh
    └── start-deploy.sh
```

### 前端页面功能
Deploy 模块前端页面（Vue 3 + Element Plus）提供以下功能：
- 统计看板：Agent 总数、运行中 Agent、Server 总数、运行中 Server
- 部署列表：展示所有部署配置
- 一键部署/停止
- 新增/编辑/删除配置
- 搜索和筛选

访问地址：`http://localhost:8090/api/deploy/`
