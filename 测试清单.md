## 测试覆盖率目标

### 核心逻辑代码（排除网络层）
| 指标 | 目标 | 说明 |
|------|------|------|
| 指令覆盖率 | >= 70% | 业务逻辑代码 |
| 方法覆盖率 | >= 80% | 公共方法 |

### 整体代码（包含网络层）
| 指标 | 目标 | 说明 |
|------|------|------|
| 行覆盖率 | >= 50% | 包含 Netty Handler 等网络层代码 |

### 排除规则
以下网络层代码因依赖外部框架难以单元测试，已排除在覆盖率验证外：
- `org.ljc.agent.handler.*` - Agent WebSocket 处理器
- `org.ljc.agent.AgentApplication` - Agent 启动类
- `org.ljc.server.handler.*` - Server WebSocket 处理器
- `org.ljc.server.ServerApplication` - Server 启动类
- `org.ljc.server.registry.*` - Server 注册服务
- `org.ljc.deploy.DeployApplication` - Deploy 启动类
- `org.ljc.deploy.config.*` - Deploy 配置类

## 测试运行命令

```bash
# 运行单元测试（build 时自动执行）
./gradlew test

# 运行 API 集成测试（需先启动服务）
./gradlew apiTest -PrunApiTests=true

# 运行特定模块测试
./gradlew :common:test
./gradlew :test:test

# 运行特定测试类
./gradlew :test:test --tests "org.ljc.test.common.CommonModelTest"

# 生成 Allure 报告
allure serve build/reports/allure-results
```
