## 1. 代码规范

### 1.1 代码风格
- 所有功能必须有接口，接口内包含详尽注释
- 必须先设计，再评审，根据代码仓现状回顾评审，再编码
- 使用Java 17及以上版本
- 使用4空格缩进
- 行长度不超过120字符

### 1.2 命名规范
- 类名: PascalCase (如: `AgentRegistry`)
- 方法名: camelCase (如: `registerAgent`)
- 常量: UPPER_SNAKE_CASE (如: `MAX_RETRIES`)
- 包名: 小写字母 (如: `org.ljc.common`)

### 1.3 注释规范
- 类注释: 说明类的用途、作者、创建日期
- 方法注释: 说明参数、返回值、异常、算法
- 行内注释: 解释复杂逻辑

## 2. 测试规范

### 2.1 测试要求
- 所有新功能必须添加测试用例
- 黑盒测试代码覆盖率需达到90%
- 所有用例必须通过才能上库

### 2.2 测试清单
| 模块 | 测试类型 | 测试用例数 | 状态 |
|------|----------|-----------|------|
| common | 单元测试 | 7 | 完成 |
| server | API测试 | 9 | 完成 |
| deploy | API测试 | 6 | 完成 |
| agent | 单元测试 | 0 | 待开发 |

### 2.3 测试报告

- 报告路径: `build/reports/allure-report`
- 运行命令: `./gradlew test` 然后 `allure serve build/reports/allure-results`

- 所有与AI的对话存储在项目的doc目录

## 6. 模块说明

### 6.1 common
公共模块，包含:
- 通信协议实体类 (Message, MessageType)
- HTTP请求/响应封装 (HttpRequest, HttpResponse)
- Agent信息实体 (AgentInfo)
- 工具类 (JsonUtil)
- 异常类

### 6.2 server
Server模块(公网部署)，包含:
- WebSocket服务器 (Netty)
- Agent注册管理
- HTTP请求转发
- 用户认证 (admin/Sys_ljc_123)
- REST API

### 6.3 agent
Agent模块(内网部署)，包含:
- WebSocket客户端
- 请求转发服务
- 心跳保活

### 6.4 deploy
部署管理模块，包含:
- Spring Boot后端
- Vue 3前端
- 部署配置管理
- 一键部署功能

### 6.5 test
测试模块，包含:
- 公共模块单元测试
- Server API黑盒测试
- Deploy API黑盒测试
- Allure报告配置

## 7. 快速开始

### 7.1 构建项目
```bash
./gradlew build
```

### 7.2 运行测试
```bash
./gradlew test
```

### 7.3 启动服务
```bash
# 启动Server
./gradlew :server:bootRun

# 启动Agent
./gradlew :agent:bootRun

# 启动Deploy
./gradlew :deploy:bootRun
```
