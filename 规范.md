## 1. 代码规范

### 1.1 代码风格
- 所有功能必须有接口，接口内包含详尽注释
- 必须先设计，再评审，根据代码仓现状回顾评审，再编码
- 使用 Java 17 及以上版本
- 使用 4 空格缩进
- 行长度不超过 120 字符

### 1.2 约定大于配置
- 所有模块不依赖配置文件即可启动（约定默认值）
- 默认端口: server=8080, deploy=8081
- 默认认证: admin/Sys_ljc_123
- 配置文件仅用于覆盖默认配置

### 1.3 命名规范
- 类名：PascalCase (如：`AgentRegistry`)
- 方法名：camelCase (如：`registerAgent`)
- 常量：UPPER_SNAKE_CASE (如：`MAX_RETRIES`)
- 包名：小写字母 (如：`org.ljc.common`)

### 1.4 注释规范
- 类注释：说明类的用途、作者、创建日期
- 方法注释：说明参数、返回值、异常、算法
- 行内注释：解释复杂逻辑

## 2. 测试规范

### 2.1 测试要求
- 所有新功能必须添加测试用例
- 核心逻辑代码覆盖率需达到 70% 以上
- 整体行覆盖率需达到 50% 以上（包含网络层代码）
- 所有用例必须通过才能上库

### 2.2 测试清单
| 模块 | 测试类型 | 测试用例数 | 状态 |
|------|----------|-----------|------|
| common | 单元测试 | 10 | 完成 |
| server | 单元测试 | 14 | 完成 |
| deploy | 单元测试 | 8 | 完成 |
| agent | 单元测试 | 40 | 完成 |
| **合计** | - | **72** | **完成** |

### 2.3 测试报告
- 使用 Allure 展示测试报告
- 报告路径：`build/reports/allure-report`
- 运行命令：`./gradlew test` 然后 `allure serve build/reports/allure-results`

### 2.4 端到端测试规范
- 每个前端界面交互必须有对应的端到端测试用例
- 使用 RestAssured 进行 HTTP API 集成测试
- 测试用例必须覆盖:
  - API 请求/响应正常场景
  - API 错误处理场景
  - 前端表单提交和数据展示
- 集成测试失败等同于 bug，必须修复
- 运行命令: 先启动服务 `./gradlew :deploy:bootRun`，再运行 `./gradlew :test:test`

### 2.5 规范审视规则
- 每次修改完成后，必须审视是否符合本规范
- 检查项:
  - [ ] 新功能是否有对应测试用例
  - [ ] 测试覆盖率是否达标
  - [ ] 提交信息格式是否正确
  - [ ] 代码是否符合命名和注释规范

## 3. 提交规范

### 3.1 提交信息格式
```
[类型] 描述

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

类型说明:
- `[特性]` - 新功能
- `[修复]` - Bug 修复
- `[优化]` - 性能优化
- `[文档]` - 文档更新

### 3.2 定期提交规则
- 每天至少提交一次（工作日）
- 每次功能完成或修复bug后立即提交
- 提交前必须通过所有测试
- 提交信息必须清晰描述变更内容

### 3.3 定期推送规则
- 每次提交后立即推送到远程仓库
- 确保远程仓库始终保持最新状态
- 推送前确认无敏感信息泄露

### 3.4 提交前检查
- [ ] 所有测试用例通过
- [ ] 核心代码覆盖率 >= 70%
- [ ] 无编译警告
- [ ] 代码格式符合规范

## 5. AI 对话存储

- 所有与 AI 的对话存储在项目的 doc 目录
- 文件大小超过 10M 则切换新文件
- 命名格式：`conversation_YYYYMMDD_N.md`

## 6. 模块说明

### 6.1 common
公共模块，包含:
- 通信协议实体类 (Message, MessageType)
- HTTP 请求/响应封装 (HttpRequest, HttpResponse)
- Agent 信息实体 (AgentInfo)
- 工具类 (JsonUtil)
- 异常类

### 6.2 server
Server 模块 (公网部署)，包含:
- WebSocket 服务器 (Netty)
- Agent 注册管理
- HTTP 请求转发
- 用户认证 (admin/Sys_ljc_123)
- REST API

### 6.3 agent
Agent 模块 (内网部署)，包含:
- WebSocket 客户端
- 请求转发服务
- 心跳保活

### 6.4 deploy
部署管理模块，包含:
- Spring Boot 后端
- Vue 3 前端
- 部署配置管理
- 一键部署功能

### 6.5 test
测试模块，包含:
- 公共模块单元测试
- Server API 黑盒测试
- Deploy API 黑盒测试
- Allure 报告配置
- JaCoCo 覆盖率配置

## 7. 快速开始

### 7.1 构建项目
```bash
./gradlew build
```

### 7.2 运行测试
```bash
./gradlew test
```

### 7.3 启动服务
```bash
# 启动 Server
./gradlew :server:bootRun

# 启动 Agent
./gradlew :agent:bootRun

# 启动 Deploy
./gradlew :deploy:bootRun
```
