# AI Forward 使用手册

## 1. 项目概述

AI Forward 是一个用于在内网部署 Agent，通过公网 Server 转发 AI 模型访问请求的系统。

### 1.1 系统架构

```
[用户] -> [公网 Server] <--WebSocket--> [内网 Agent] -> [AI 模型]
```

- **Server**: 部署在公网，接受 Agent 注册，转发用户 HTTP 请求到内网 Agent
- **Agent**: 部署在内网，连接 Server，将请求转发到本地 AI 模型
- **Deploy**: 部署管理模块，提供 Web 界面管理 Agent 和 Server

### 1.2 模块说明

| 模块 | 说明 | 端口 |
|------|------|------|
| server | 公网转发服务 | HTTP: 8080, WebSocket: 8888 |
| agent | 内网 AI 转发客户端 | - |
| deploy | 部署管理 Web 界面 | 8081 |

## 2. 快速开始

### 2.1 构建项目

```bash
./gradlew build -x test
```

构建产物位于各模块的 `build/distributions/` 目录：
- `agent/build/distributions/agent-1.0.0.tar`
- `server/build/distributions/server-1.0.0.tar`
- `deploy/build/distributions/deploy-1.0.0.tar`

### 2.2 解压部署

```bash
# 解压各模块
tar -xzf agent-1.0.0.tar
tar -xzf server-1.0.0.tar
tar -xzf deploy-1.0.0.tar
```

## 3. Server 部署

### 3.1 约定配置（无需配置即可启动）

Server 采用约定大于配置，无需配置文件即可启动：

```bash
cd server-1.0.0
./bin/server
```

默认配置：
- HTTP 端口: 8080
- WebSocket 端口: 8888
- 认证用户: admin
- 认证密码: Sys_ljc_123

### 3.2 自定义配置（如需覆盖默认配置）

如有需要，可创建 `config/application.yml` 覆盖默认配置：

```bash
cd server-1.0.0
./bin/server
```

或使用 Spring Boot：

```bash
java -jar server-1.0.0.jar
```

### 3.3 验证

```bash
# 健康检查
curl http://localhost:8080/api/server/health

# 认证测试
curl -u admin:Sys_ljc_123 http://localhost:8080/api/server/agents
```

## 4. Agent 部署

### 4.1 环境变量配置

Agent 通过环境变量配置：

| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| SERVER_HOST | Server 地址 | localhost |
| SERVER_PORT | Server WebSocket 端口 | 8888 |
| SERVER_WS_PATH | WebSocket 路径 | /ws |
| AGENT_ID | Agent 标识 | auto-generated |
| AGENT_NAME | Agent 名称 | AI Agent |
| ALLOWED_TARGETS | 允许转发的目标 URL | * |
| CONNECT_TIMEOUT | 连接超时(毫秒) | 5000 |
| SSL_ENABLED | 是否启用 SSL | false |

### 4.2 启动

```bash
cd agent-1.0.0

# 方式1: 使用环境变量
export SERVER_HOST=your-server-ip
export SERVER_PORT=8888
./bin/agent

# 方式2: 使用命令行参数
./bin/agent --server.host=your-server-ip --server.port=8888
```

### 4.3 Docker 部署

```dockerfile
FROM openjdk:17-slim
COPY agent-1.0.0.tar /app/
WORKDIR /app
RUN tar -xzf agent-1.0.0.tar
ENV SERVER_HOST=your-server-ip
ENV SERVER_PORT=8888
ENTRYPOINT ["./bin/agent"]
```

## 5. Deploy 部署（可选）

Deploy 模块提供 Web 界面管理 Agent 和 Server。

### 5.1 约定配置（无需配置即可启动）

Deploy 采用约定大于配置，默认使用嵌入式 H2 数据库：

```bash
cd deploy-1.0.0
./bin/deploy
```

默认配置：
- HTTP 端口: 8081
- 数据库: 嵌入式 H2（内存）

### 5.2 一键部署功能

Deploy 包已内置 Agent 和 Server，解压后可直接通过 Web 界面一键部署：

1. 启动 Deploy：`./bin/deploy`
2. 打开浏览器访问：http://localhost:8081
3. 点击"新建配置"添加 Agent 或 Server
4. 点击"部署"按钮一键启动

### 5.3 自定义配置（如需覆盖默认配置）

如有需要，可创建 `config/application.yml`：

```bash
cd deploy-1.0.0
./bin/deploy
```

### 5.3 访问

打开浏览器访问：http://localhost:8081

## 6. 使用示例

### 6.1 通过 Server 转发 AI 请求

假设：
- Server 部署在 `203.0.113.10`，端口 8080
- Agent 连接了本地 AI 模型（Ollama）

```bash
# 通过 Server 访问内网 Ollama
curl -X POST http://203.0.113.10:8080/api/proxy/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "model": "llama2",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### 6.2 指定 Agent 转发

```bash
# 指定 agent-id 进行转发
curl -X POST http://203.0.113.10:8080/api/proxy/v1/chat/completions?agentId=agent-001 \
  -H "Content-Type: application/json" \
  -d '{"model": "llama2", "messages": [{"role": "user", "content": "Hello!"}]}'
```

### 6.3 认证方式

支持两种认证方式：

1. **Basic Auth**:
```bash
curl -u admin:Sys_ljc_123 http://203.0.113.10:8080/api/server/agents
```

2. **Bearer Token**:
```bash
# 先获取 token
TOKEN=$(curl -s -u admin:Sys_ljc_123 http://203.0.113.10:8080/api/server/login | jq -r '.token')

# 使用 token
curl -H "Authorization: Bearer $TOKEN" http://203.0.113.10:8080/api/server/agents
```

## 7. API 参考

## 8. SSL/TLS 配置

### 8.1 生成密钥库

```bash
# 使用 keytool 生成密钥库
keytool -genkeypair \
  -alias server \
  -keyalg RSA \
  -keysize 2048 \
  -validity 365 \
  -keystore keystore.jks \
  -storepass your-password
```

### 8.2 配置 Server SSL

```yaml
server:
  ssl-enabled: true
  key-store: classpath:keystore.jks
  key-store-password: your-password
```

### 8.3 配置 Agent SSL

```bash
export SSL_ENABLED=true
export SERVER_WSS_URL=wss://your-server:8888/ws
```

## 9. 故障排除

### 9.1 Agent 无法连接 Server

1. 检查 Server 是否运行：`curl http://server:8080/api/server/health`
2. 检查防火墙是否开放 8888 端口
3. 检查 SSL 配置是否匹配
4. 查看日志确认错误信息

### 9.2 请求转发失败

1. 确认 Agent 已连接：`curl http://server:8080/api/server/agents`
2. 检查目标 AI 模型是否正常运行
3. 查看 Agent 日志

### 9.3 认证失败

1. 确认用户名密码正确
2. 检查 Token 是否过期
3. 确认请求头格式正确

## 10. 性能优化

### 10.1 Agent 优化

- 调整 `CONNECT_TIMEOUT` 适应网络环境
- 合理设置 `ALLOWED_TARGETS` 限制转发范围
- 启用 SSL 加密提高安全性

### 10.2 Server 优化

- 调整心跳检测间隔：`heartbeat-interval`
- 调整最大空闲时间：`max-idle-time`
- 使用 SSL 加密 WebSocket 连接

## 11. 常见问题

**Q: 如何查看 Agent 状态？**
A: 访问 `http://server:8080/api/server/agents`

**Q: 支持哪些 AI 模型？**
A: 任何支持 HTTP API 的模型都可以，如 OpenAI、Ollama、LocalAI 等

**Q: 如何实现高可用？**
A: 可以部署多个 Agent 连接到同一个 Server，或部署多个 Server 实现负载均衡

**Q: 如何监控？**
A: 可以通过 Deploy 模块的 Web 界面查看状态，或集成 Prometheus + Grafana
